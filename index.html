SCAI - Multi-modal Conversational AI

SCAI, a conversational AI designed by identifying the gap in current conversational assistants like Apple's Siri and Amazon's Alexa. These assistants operate more like utility assistants with limitations such as; operating in single modality no persistent memory for continuous conversation and they lack personal human interaction.
With the introduction of SCAI, this bridges these gaps by setting core objectives such as;

Multi-modal capabilities: This combines text, voice, vision as integral components for personal interaction. The integration of computer vision, specifically emotion recognition and facial recognition aids in providing SCAI with context awareness, making SCAI's responses more personal, more human-like. Essentially, the vision is in a sense comparative to some functions of the human eyes.

Adaptive learning: SCAI is designed to remember users preferences or instructions and tailor its responses accordingly. For example: In response to SCAI, I say I prefer short but informative answers.B efore adaptation, it may give long paragraphs but with its adaptive capabilities it would give a more concise summary. It changes its behaviour based on my preference. The users preferences are also used to develop the SCAI capabilities.

Autonomous learning: SCAI is designed to use its understanding of the user and their preferences to tailor its responses and actions. For instance, I say I like listening to classical music when I am in a bad mood and next time SCAI notices I'm in a bad mood (using its vision component), it recommends I listen to classical music to make me feel better. It uses its knowledge of me to tailor its actions/suggestions.

SCAI stands out as it is designed to be able to initiate conversations with the user.



Future Work

There is more work to be done in developing SCAI. Some features i would like to implement includes;

  • Graceful fallback, this ensures that when a modality isn't functioning, users are redirected and recommended to use a different one
  • Search functionality, this feature allows users to search conversation history for keywords/chats they would like to find and they are taking to the location of keyword
  • Consistent comprehensive testing to ensure working functionality
  • Live transcription for voice conversations which is displayed to users
  • Interruption handling during speech for a more natural conversation flow
  • Semantic memory
  • Integration of IoT

There are more innovative ideas I would like to implement for this amazing project but the current priority is making SCAI fully functional and operational as it is currently facing some technical issues.

WordMatrix

WordMatrix is an engaging word puzzle game that challenges players to unscramble scrambled words within limited attempts. Built with Python object-oriented programming, the game features:

Intelligent Word Selection: Curated vocabulary ranging from common to advanced words
Progressive Difficulty: Adaptive challenges that scale with player skill
Strategic Gameplay: Three attempts per word with scoring system
Session Management: Track used words to ensure variety across gameplay sessions

The game demonstrates strong software architecture principles with modular class design separating display logic, word management, and game mechanics.
